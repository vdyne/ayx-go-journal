# ayx-go-journal

本仓库用于归档和发布多个独立的 HTML 页面。这些页面内容彼此独立，不针对任何特定域名或具体网站，仅作为静态页面集合进行维护与展示。

## 项目简介

`ayx-go-journal` 是一个纯粹的静态页面归档仓库。仓库中的每个 HTML 文件代表一个独立的条目，可用于记录、展示或引用各种信息。项目本身不包含后端逻辑或动态内容生成，所有页面均为预先生成的静态文件。

## 目录说明

```
/
├── README.md          # 本文件
├── index.html         # 可选：首页或导航页
├── pages/             # 页面归档目录
│   ├── page-001.html
│   ├── page-002.html
│   └── ...
└── assets/            # 静态资源目录（样式、脚本、图片等）
    ├── css/
    ├── js/
    └── images/
```

- `pages/`：存放所有独立的 HTML 页面文件，按编号或时间顺序归档。
- `assets/`：存放页面引用的公共资源文件，便于统一管理和复用。

## 页面归档说明

- 每个 HTML 页面均为自包含的独立文件，可单独访问。
- 页面内容不限定主题或领域，仅作为信息载体存在。
- 归档遵循“一个文件一条记录”的原则，便于检索与引用。
- 页面命名建议采用语义化或顺序化方式，以保持整洁。

## 维护说明

- 新增页面：在 `pages/` 目录下创建新的 HTML 文件，并确保其内部链接与资源引用正确。
- 更新页面：直接编辑对应 HTML 文件，注意保持页面结构与样式一致。
- 资源管理：若页面使用公共资源，请将资源文件放入 `assets/` 对应子目录，并在 HTML 中引用相对路径。
- 本仓库不提供自动化部署脚本，所有页面更新后需手动提交并推送至 GitHub。
